# River Club iOS UI 设计规格

日期：2026-07-13  
状态：UI 已由用户确认，等待规格文档复核  
目标设备：iPhone 16 Pro Max  
屏幕方向：全程横屏

## 1. 产品边界

River Club 是一款使用无现金价值虚拟筹码的休闲德州扑克游戏。首版只支持无限注德州扑克，不包含真钱充值、提现、现金兑换或其他可变现入口。

产品借鉴成熟线上扑克产品的信息架构和常见交互习惯，但使用原创品牌、名称、图形、配色细节与界面素材，不复制 Replay Poker 的商标、Logo、插画或专属视觉资产。

## 2. 首版 UI 范围

首轮包含以下 7 个核心界面：

1. 启动 / 登录
2. 游戏大厅
3. 牌桌列表
4. 买入确认弹窗
5. 九人游戏牌桌
6. 锦标赛列表
7. 个人中心

任务、排行榜、商城、好友、私聊等扩展页面不进入首轮 UI 与功能范围。

## 3. 视觉方向

整体采用“经典赌场”方向：

- 主背景：深墨绿和近黑绿，保证长时间浏览时不过亮。
- 主交互色：古金色，用于主按钮、选中状态、当前行动玩家和关键筹码数字。
- 辅助材质：少量胡桃木棕，用于牌桌边框和空间层次。
- 文字：主要文字为暖白，次要文字为低饱和灰绿。
- 圆角：页面组件使用克制的中等圆角，避免娱乐化过强。
- 阴影：只用于弹窗、牌桌与当前行动状态，不堆叠装饰性阴影。
- 品牌工作名：River Club；正式名称和 Logo 可在品牌阶段替换。

## 4. 横屏与安全区

- 以 iPhone 16 Pro Max 横屏逻辑尺寸约 956 × 440 pt 为设计基准。
- 全应用锁定横屏，不在大厅与牌桌之间切换方向。
- 左侧避让灵动岛区域，右侧避让 Home Indicator 区域。
- 系统安全区内不放退出、下注、确认等关键操作。
- 紧凑宽度下保持核心操作可见；非关键统计可隐藏或折叠。

## 5. 导航架构

登录后使用固定左侧导航：

- 游戏大厅
- 锦标赛
- 我的牌局
- 个人中心

进入游戏牌桌后隐藏主导航，切换为沉浸式全屏。退出牌桌时返回之前的大厅或牌桌列表位置。

核心路径：

`启动/登录 → 游戏大厅 → 牌桌列表 → 买入确认 → 游戏牌桌`

辅助路径：

- `游戏大厅 → 快速加入 → 买入确认 → 游戏牌桌`
- `左侧导航 → 锦标赛 → 报名或查看状态`
- `左侧导航 → 个人中心 → 战绩、账户或设置`

## 6. 页面规格

### 6.1 启动 / 登录

- 左侧为品牌、欢迎语和登录操作，右侧为原创扑克氛围图形。
- 主操作为“使用 Apple 登录”。
- 次操作为“游客快速体验”。
- 底部显示用户协议、隐私政策和年龄提示。
- 不展示现金、提现、奖金兑换或收益暗示。

### 6.2 游戏大厅

- 左侧固定主导航。
- 顶部显示欢迎信息、通知入口和虚拟筹码余额。
- 内容区包含“为你推荐、常规牌桌、已收藏、新手专区”标签。
- 推荐桌卡片突出玩法、盲注、在线人数与立即入桌按钮。
- 快速加入允许选择常用盲注档位并自动匹配空位。
- 热门桌列表展示牌桌名、盲注、玩家数、平均底池与加入状态。

### 6.3 牌桌列表

- 提供全部、低盲注、中盲注、高盲注和收藏筛选。
- 支持按桌型、是否有空位和盲注范围进一步过滤。
- 每行展示牌桌名、盲注、当前人数、平均底池和状态。
- 有空位显示“加入”，满桌显示“候补”。

### 6.4 买入确认

- 以居中模态弹窗覆盖列表，背景轻度变暗。
- 显示牌桌名称、玩法和盲注。
- 使用滑杆选择允许范围内的虚拟筹码买入量。
- 提供低于阈值时自动补充的开关。
- 主按钮为“确认买入并入座”。
- 余额不足时不关闭弹窗，显示清晰提示并提供返回操作。

### 6.5 九人游戏牌桌

- 牌桌显示 8 位对手和 1 位本人，所有座位完整可见且互不遮挡。
- 本人座位固定在屏幕下方视觉中心；头像固定为正圆，不参与横向压缩。
- 公共牌置于牌桌中心。
- 底池数值置于公共牌下方，筹码堆位于底池下方。
- 当前行动玩家使用古金色描边和倒计时状态。
- 右下角为弃牌、跟注和加注三个主操作。
- 加注区提供半池、四分之三池等预设比例和连续滑杆。
- 左下角提供牌桌聊天和表情入口。
- 顶部只保留返回、牌桌信息、筹码余额和设置。

### 6.6 锦标赛列表

- 标签包括即将开始、已报名、进行中和已结束。
- 卡片区分新手免费赛、经典赛事与快速赛事。
- 每张卡片显示开始时间、报名人数、人数上限、奖池和报名状态。
- 免费赛事明确显示“免费报名”，不使用现金符号。

### 6.7 个人中心

- 展示头像、昵称、会员级别和经验进度。
- 核心娱乐统计包括总手数、入池率和赛事奖励次数。
- 提供牌局记录、成就徽章、账户与安全、声音与震动入口。
- 不展示现金账户或可提现余额。

## 7. 组件边界

后续 SwiftUI 实现应优先复用以下独立组件：

- `AppSidebar`：横屏主导航与用户摘要。
- `ChipBalancePill`：虚拟筹码余额和补充入口。
- `TableRow`：牌桌列表数据行。
- `FeaturedTableCard`：大厅推荐牌桌。
- `BuyInSheet`：买入金额和自动补充设置。
- `PokerSeatView`：头像、昵称、筹码与行动状态。
- `PlayingCardView`：手牌与公共牌。
- `BetControlBar`：比例预设、滑杆和下注金额。
- `PokerActionButtons`：弃牌、跟注、加注。
- `TournamentCard`：赛事信息和报名状态。
- `ProfileStatCard`：个人统计。

每个组件只接收展示所需的数据和事件回调，不直接持有网络或游戏逻辑。

## 8. 界面状态与异常处理

每个列表页面需要定义加载中、空数据、离线和重试状态：

- 加载中：使用低干扰占位骨架，不阻塞左侧导航。
- 空数据：说明筛选条件下没有结果，并提供清除筛选入口。
- 离线：顶部显示非模态离线条；已在牌桌中时保留当前画面并显示重连状态。
- 加入失败：保留用户所在列表位置并说明原因。
- 余额不足：买入弹窗内提示可用筹码和最低买入要求。
- 牌桌满员：将加入操作替换为候补或返回列表。
- 操作超时：牌桌显示自动弃牌或过牌结果，避免只靠短暂 Toast。

## 9. 可访问性与交互规则

- 关键触控区域目标不小于 44 × 44 pt。
- 颜色不作为状态的唯一表达；行动状态同时使用文字和描边。
- 支持系统减少动态效果设置。
- 牌面花色使用颜色与符号共同区分。
- 关键数字使用等宽数字，避免筹码变化造成布局跳动。
- 游戏牌桌优先保证信息不重叠，其次才是装饰细节。

## 10. UI 验收标准

- iPhone 16 Pro Max 横屏下不存在被灵动岛或 Home Indicator 遮挡的关键内容。
- 九人桌严格显示 9 个座位：8 位对手和 1 位本人。
- 本人头像在所有昵称和筹码长度下保持正圆。
- 底池、公共牌和筹码堆互不重叠。
- 右下角三个主要操作在常见下注金额长度下不截断。
- 所有页面使用相同的颜色、字体层级、圆角和间距规则。
- 用户可以从登录进入大厅、选桌、买入并到达牌桌，不出现断链页面。
- 全部虚拟筹码显示均不使用真实货币符号。
