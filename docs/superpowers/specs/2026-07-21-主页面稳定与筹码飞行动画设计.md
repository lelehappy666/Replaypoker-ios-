# 主页面稳定与筹码飞行动画设计

## 目标

- 游戏大厅、锦标赛、我的牌局和个人中心互相切换时，左侧导航、右侧内容区域和整体背景不发生位移或缩放。
- 下注、筹码归池、退回未跟注筹码和结算派彩均表现出多组筹码连续飞动的感觉。
- 右上角账户筹码与金额采用效果图中的立体堆叠胶囊布局，不遮挡牌桌座位或手牌。
- 保持现有牌局规则、机器人身份、金额计算和结算顺序不变。

## 主页面固定外壳

采用固定坐标外壳方案。

- 外层使用设备可用画布作为唯一尺寸来源。
- 左侧导航栏使用固定宽度，并固定在安全区内的同一坐标。
- 导航栏与右侧内容之间使用固定间距。
- 右侧内容宽度由画布宽度减去安全边距、导航栏宽度和间距得到，不再依赖页面内容的理想尺寸。
- 四个主页面仅替换右侧内容；外壳、导航栏和背景承载层保持同一结构与身份。
- 页面切换禁用外壳位移、缩放和隐式过渡动画；选中菜单颜色仍立即更新。
- 各页面内部可以继续加载数据，但加载态与内容态必须占用相同的右侧固定画布。

## 筹码飞行动画

### 动画表现

每次筹码移动使用四组小型筹码簇，而不是逐颗创建大量视图：

- 四组筹码簇沿相近但不完全重合的弧线路径移动。
- 每组依次延迟约 0.05 秒，营造筹码连续飞出或飞入的感觉。
- 每组使用轻微不同的横向偏移、缩放和旋转，避免像一整块筹码平移。
- 动画结束后一次性隐藏临时筹码层，实际金额仍以牌局状态为准。

### 方向

- 玩家下注：从玩家座位附近飞向该玩家的桌前下注位置。
- 本轮下注归池：从桌前下注位置飞向中央底池。
- 未跟注筹码退回：从桌前下注位置飞回对应玩家座位。
- 单人独赢：中央底池的四组筹码一次连续飞向赢家。
- 多人获胜：按现有结算事件顺序，每位赢家各播放一次完整筹码簇动画；每次显示该赢家实际获得的金额。

### 动态效果设置

系统开启“减少动态效果”时仍保留筹码分批飞动，不允许瞬移。此模式缩短持续时间并减小弧线与旋转幅度，兼顾用户要求与视觉舒适度。

## 右上账户筹码组件

- 使用固定宽高的深绿色半透明胶囊。
- 左侧为红、紫、绿、黑四种真实赌场配色的立体筹码堆，筹码上下堆叠并彼此遮叠。
- 右侧为完整金额，使用金色等宽数字。
- 筹码堆和数字各自拥有固定区域，不互相覆盖。
- 组件固定在顶部保留区域，设置按钮位于其右侧；两者不得进入任何座位手牌框。
- 组件按牌桌参考画布统一缩放，内部不再进行二次自适应缩放。

## 代码边界

- 主页面尺寸计算集中在公共外壳中，各业务页面不再自行影响外壳宽度。
- 筹码飞行路径和分组进度集中在牌桌布局与动画展示模型中。
- 飞行筹码只负责视觉展示，不修改牌局状态和筹码账本。
- 账户余额胶囊使用独立的牌桌组件，不改变座位筹码和底池筹码组件。

## 异常与生命周期处理

- 新动画事件到达时取消上一次未完成的临时动画，避免筹码残留。
- 页面离开、牌局暂停或视图销毁时清理动画任务。
- 金额为零的结算事件不生成可见筹码簇。
- 无法取得座位或下注位置时，不显示飞行动画，但不阻塞牌局流程。

## 验证范围

按用户要求不增加大量 UI 自动化测试。

- 对固定外壳尺寸计算和筹码分组进度增加少量纯逻辑检查。
- 完成 Swift 编译检查，确保四个主页面与牌桌视图可正常构建。
- 最终视觉效果由用户在 iPhone 16 Pro Max 横屏模拟器中检查。
