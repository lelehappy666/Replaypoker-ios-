# 牌桌筹码联动、主页面稳定与音效设计

## 目标

本次修改解决以下四项问题：

1. 下注、退回筹码和派彩过程中，牌桌上的筹码堆高度必须随实际动画金额同步升高或降低。
2. 游戏大厅、锦标赛、我的牌局和个人中心切换时，不再出现轻微位移、缩放或异步加载造成的跳动。
3. 牌桌右上角账户余额区保持完整清晰，同时不能遮挡顶部座位、头像或手牌。
4. 牌桌增加轻量、合适且可关闭的本地音效。

## 根因

### 筹码堆高度没有联动

当前飞行筹码是独立覆盖层。下注区、底池和座位筹码仍直接读取已完成状态，而且 `CasinoChipPileView` 的每列高度由固定序号决定，没有使用动画进度计算临时金额。因此画面只显示筹码飞行，起点和终点筹码堆不会逐层变化。

### 主页面仍有轻微跳动

当前四个页面虽然共用固定外框尺寸，但路由切换仍会重建背景和右侧页面内容。页面各自的加载状态、内部几何布局和背景资源会在首帧采用不同的理想尺寸，随后再稳定到最终布局，从而产生轻微位移。

### 右上余额区占位偏大

账户筹码图、金额和设置按钮分别占用较宽区域，筹码图自身层叠高度也较大。顶部座位靠近右侧时，余额区虽然裁切了自身内容，但整体仍会侵入座位视觉区域。

## 设计方案

### 一、动画金额模型

为当前动画事件计算三类临时显示金额：

- 起点金额：随飞行动画进度逐步减少。
- 终点金额：随飞行动画进度逐步增加。
- 飞行金额：按四组错峰筹码的平均进度计算，保证筹码堆变化与飞行画面一致。

事件对应关系：

- 下盲注、下注进入下注区：玩家座位筹码降低，下注区筹码升高。
- 街道筹码进入底池：下注区筹码降低，底池筹码升高。
- 未跟注筹码退回：下注区筹码降低，玩家座位筹码升高。
- 派彩：底池筹码降低，赢家座位筹码升高。

金额显示始终使用整数，并在动画结束时精确回到协调器提供的真实状态。动画只影响展示，不修改游戏规则、账户或牌局数据。

### 二、筹码堆高度规则

`CasinoChipPileView` 根据显示金额计算可见筹码层数和列数：

- 金额为零时不显示筹码。
- 小金额显示较矮的单列或双列。
- 金额增加时逐步增加每列层数，再增加列数。
- 设置最大可见层数，避免大底池无限增高。

动画过程中使用临时显示金额重新计算堆高，并使用短时平滑过渡。数字金额与筹码堆同步变化，不提前跳到最终值。

### 三、四个主页面稳定外壳

四个主页面共用一个常驻的 `SidebarPageShell`：

- 侧栏实例不随路由销毁。
- 背景层使用固定画布与统一裁切规则，背景切换不参与外壳尺寸计算。
- 四个右侧页面放入同一固定内容区域，通过可见性和命中状态切换，不使用会改变父布局的条件重建。
- 页面加载态、空态和已加载态都占满同一个右侧固定区域。
- 所有坐标对齐到设备像素，消除缩放后半像素带来的轻微抖动。

切换页面时不使用位移或缩放动画，只更新选中状态和右侧可见内容。

### 四、右上账户余额区

余额区改为固定的小型胶囊：

- 使用更紧凑的三组赌场筹码堆图形。
- 金额采用固定宽度、单行、等宽数字。
- 余额胶囊和设置按钮共同放入顶部安全区固定框。
- 胶囊自身严格裁切，且与顶部右侧座位保持固定安全间距。
- 大额数字只缩小字体，不扩大胶囊宽度。

### 五、牌桌音效

新增本地牌桌音效服务，使用 `AVFoundation` 在设备内生成并播放短音效，不访问网络，也不依赖系统私有音效编号。

音效类型：

- 发底牌与公共牌：轻微纸牌滑动声。
- 下注、下盲注、退回筹码、筹码进入底池：短促筹码碰撞声。
- 轮到真人玩家行动：克制的提示音，每轮只播放一次。
- 单人或多人派彩：短促获胜音，派彩筹码动画仍分别对应每位赢家。

声音默认开启。在“个人中心 → 声音与震动”中提供总开关，并使用本地持久化保存选择。关闭后所有牌桌音效立即停止，重新进入应用后仍保持关闭状态。音效不会阻塞牌局流程，播放失败也不会影响操作或结算。

## 边界与异常处理

- 动画金额使用范围限制，不能出现负数或超过事件前后真实总额。
- 动画事件缺少座位或金额为零时，不显示飞行层，也不播放对应音效。
- 页面处于后台、牌桌暂停或视图离开时，停止正在播放的音效并取消动画任务。
- 开启系统“减少动态效果”时，仍保留用户要求的分层筹码飞行和堆高变化，但缩短持续时间和弧线距离。
- 声音开关与系统静音策略保持一致，不强制覆盖用户设备音频设置。

## 验证范围

只做与本次修改直接相关的必要验证：

- 动画临时金额在开始、中间和结束进度下正确守恒。
- 四个主页面使用同一固定外壳和像素对齐坐标。
- 声音开关默认开启、可持久保存且关闭后不触发播放请求。
- 项目能够完成 iOS 模拟器编译。

不增加大规模 UI 自动化测试，最终视觉效果由用户在 iPhone 16 Pro Max 横屏模拟器中确认。
