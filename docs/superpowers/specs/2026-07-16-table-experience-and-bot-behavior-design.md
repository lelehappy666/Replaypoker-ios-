# River Club 牌桌体验与机器人行为修复设计

日期：2026-07-16  
状态：设计已确认

## 1. 目标

本项目修复 iPhone Pro Max 横屏大厅与九人现金桌的可用性问题，并纠正机器人连续全下的不合理决策。

完成后应达到：

- 大厅左侧导航占用更少横向空间。
- 本人手牌和公共牌清晰可读。
- 牌桌实时展示本人当前牌型。
- 机器人依据公开信息、牌力、底池赔率、有效筹码和设置做出合理动作，不把全下当作普通随机动作。
- 结算时明确展示赢家和筹码移动方向。
- 九个座位头像样式一致。
- 牌桌提供安全、可确认、资金守恒的离桌入口。

## 2. 本期范围

本期包含：

- 大厅侧栏横屏宽度与间距调整。
- 牌桌手牌、公共牌和九座位布局调整。
- 本人实时牌型提示。
- 机器人下注尺度与全下候选规则修复。
- 单赢家、多人平分和边池奖励动画。
- 头像圆形视觉统一。
- 牌桌左上角离桌按钮、二次确认、自动弃牌、后台结算和返回来源页面。
- 对应单元测试、属性测试、协调器测试、UI 测试和通用 iOS 构建验收。

本期不包含：

- 新增扑克变体。
- 修改德州扑克胜负、边池或筹码守恒规则。
- 为机器人开放任何隐藏牌、牌堆、种子或其他玩家观察。
- 重新设计全部页面或替换现有牌局核心。
- 网络对战、服务器结算或真钱功能。

## 3. 大厅侧栏

`AppSidebar` 当前固定宽度为 `220pt`，在横屏手机上占用过多主内容空间。本期在 iPhone 横屏固定为 `168pt`，水平内边距从 `16pt` 调整为 `10pt`。

要求：

- 文字、SF Symbol 图标和当前选中状态继续完整显示。
- 每个入口的最小触控高度不低于 `44pt`。
- 大字号模式下不截断为不可理解的单字。
- 主内容获得的额外宽度用于列表、推荐牌桌和快速加入控件，不增加无意义留白。
- iPad 或更宽画布可以在布局令牌允许时适度放宽，但 iPhone 横屏不回到 `220pt`。

## 4. 牌桌牌面与座位布局

### 4.1 牌面尺寸

尺寸下限：

- 公共牌不得小于 `46×62pt`。
- 本人手牌不得小于 `42×57pt`。
- 机器人手牌不得小于 `34×46pt`，并根据九座位安全区域使用独立尺寸，不得挤入顶部栏、聊天区或下注区。

`TableCardView` 继续保持统一纵横比。布局通过牌桌令牌和安全区域计算，不在多个视图中重复硬编码。

### 4.2 座位

九个座位必须在 iPhone Pro Max 横屏安全画布内完整显示。底部本人座位保持视觉中心，右侧下注区和左下聊天区不得覆盖座位。

牌面放大后需要重新计算：

- 顶部三座位纵向位置。
- 左右边缘座位与屏幕圆角、灵动岛安全区的距离。
- 底部三座位与操作区、聊天区的间距。
- 中央公共牌区域的宽高。

### 4.3 头像

所有座位始终显示固定尺寸的圆形头像底板：

- 本人：金色强调圆形。
- 机器人：深绿色或现有高层表面色圆形。
- 弃牌、全下、当前行动和赢家状态只能改变透明度、描边或状态徽标，不能移除圆形底板。
- 名称与头像分离布局，避免文字覆盖圆形。

## 5. 实时牌型提示

牌型提示位于公共牌下方，采用用户确认的方案 A。

### 5.1 显示规则

- 翻牌前：显示起手牌描述，例如 `起手牌：K7 不同花`、`起手牌：AA 对子`。
- 翻牌后：使用本人两张手牌与已经公开的公共牌计算当前最佳五张牌型。
- 显示中文牌型名称，例如：
  - 高牌 A
  - 一对 9
  - 两对，K 和 9
  - 三条 6
  - 顺子，10 高
  - 同花，A 高
  - 葫芦，Q 带 6
  - 四条 9
  - 同花顺，K 高
- 牌型提示下方依次显示底池金额和底池筹码堆。

### 5.2 安全边界

展示模型只能接收：

- 本人已经公开给本人的两张手牌。
- 当前已经公开的公共牌。

不得接收机器人底牌、牌堆、随机种子、完整恢复检查点或存档中的最终底牌。牌型提示模块不得被机器人决策模块反向依赖。

## 6. 机器人决策修复

### 6.1 根因

当前机器人存在两个相互放大的问题：

- `allIn` 长期作为普通合法候选参与随机加权，即使是深筹码和普通牌力。
- 普通下注与加注尺寸在“最小合法金额”和“全部剩余筹码”之间直接按设置百分比插值，默认下注尺度也可能产生接近整副筹码的下注。

这会造成首个过大下注触发后续机器人只能大额跟注或全下，最终出现多人同时全下。

### 6.2 下注尺寸

下注和加注改为以底池与盲注为尺度：

- 从最小合法下注或最小合法加注开始。
- 把机器人 `betSizing` 的 `0...100` 线性映射到 `33%...100%` 当前底池；默认值 `50` 对应 `66.5%` 底池，最终筹码金额向下取整后受合法范围约束。
- 结果受最大合法金额和有效筹码约束。
- 普通深筹码场景不因为最大合法金额等于全部筹码就自动趋近全下。
- 金额达到真实最大合法金额时仍使用规则核心的全下动作映射，确保合法性。

### 6.3 全下资格

全下不再无条件加入候选集合。只有满足至少一种资格时才参与决策：

- 剩余筹码小于正常跟注金额，只能短筹码跟注全下。
- 有效筹码不高于 `12` 个大盲。
- 综合决策强度不低于 `8,500/10,000`。
- 困难模式模拟有效胜率不低于 `8,000/10,000`。
- 激进模型同时满足：综合决策强度不低于 `7,500/10,000`、有效筹码不高于 `30` 个大盲、跟注底池赔率不高于 `40%`。

弱牌或中等牌力的深筹码机器人不能仅因为随机种子、激进度或下注尺度而全下。

### 6.4 设置兼容

以下设置继续有效：

- 难度。
- 决策模型。
- 激进度。
- 诈唬频率。
- 跟注范围。
- 下注尺度。
- 思考速度。
- 是否分析公开历史。

设置只调整合理候选的权重和尺寸，不能绕过全下资格或访问隐藏信息。

## 7. 赢家与筹码动画

牌局核心已经发布 `awardPot(seat:amount:potIndex:)` 和 `highlightWinner` 事件。本期继续以这些事件作为唯一动画数据源。

### 7.1 动画流程

每个底池奖励按核心事件顺序展示：

1. 中央底池筹码堆高亮。
2. 显示 `玩家名赢得 3,600`。
3. 筹码图形从中央底池位置移动至赢家头像或筹码区域。
4. 对应赢家座位显示金色描边和轻微强调。
5. 更新最终筹码显示。

多人平分或存在边池时，按 `potIndex` 和奖励事件依次移动到各赢家，不能把全部筹码只动画给一个玩家。

### 7.2 减少动态效果

系统开启“减少动态效果”时：

- 不执行长距离飞行动画。
- 直接高亮赢家并显示奖励文字。
- 事件顺序、金额和最终筹码保持完全一致。

UI 动画取消或视图退出不能取消牌局核心结算。

## 8. 离桌流程

### 8.1 入口

牌桌左上角显示 `离桌` 按钮，与牌桌名称同处顶部栏。设置按钮和余额继续位于右上角。

### 8.2 二次确认

点击后显示覆盖整个牌桌的模态确认：

- 标题：`确认离开牌桌？`
- 说明：`当前手牌将自动弃牌，结算完成后带走剩余筹码。`
- 操作：`继续游戏`、`弃牌并离桌`

背景禁止交互并从辅助功能树隐藏，确认按钮具备危险操作语义。

### 8.3 执行

确认后：

1. 禁止重复点击和牌桌操作。
2. 取消未完成的机器人决策任务和玩家动作请求。
3. 如果手牌进行中，使用专用协调器离桌流程合法标记真人弃牌。
4. 机器人在后台加速完成剩余合法动作与结算，不展示冗长行动动画。
5. 原子保存完成记录并将真人剩余筹码退回余额。
6. 清理当前桌状态。
7. 返回进入牌桌前的页面；默认来源为大厅。

不得通过伪造当前行动者或普通玩家动作接口执行越权弃牌。

### 8.4 错误和幂等

- 同一次离桌使用稳定业务标识，重复确认不能重复退款。
- 保存或结算失败时不离开页面、不清空协调器、不重复扣款。
- 页面显示中文错误和 `重试离桌`。
- 应用进入后台后恢复时继续从持久化状态对齐结果。

## 9. 模块边界

新增或调整以下小型模块：

- `PokerTableLayout`：侧栏、座位、牌面和中心区域布局令牌。
- `CurrentHandPresentation`：起手牌和正式牌型中文展示。
- `BotDecisionEngine` / `RuleBasedEvaluator`：全下资格与候选权重。
- 下注尺寸策略：把底池比例转换为合法筹码金额。
- `TableAnimationPresentation`：根据底池中心与座位位置生成奖励动画展示状态。
- `CashTableCoordinator`：专用离桌流程和加速结算。
- `AppSession`：返回来源页面、离桌确认状态、错误与重试。

结算、边池、筹码守恒和存档仍由现有 `PokerCore`、`PokerSession` 与协调器负责。

## 10. 测试与验收

### 10.1 布局与展示

- iPhone Pro Max 横屏侧栏宽度明显小于旧 `220pt`，所有入口仍可读可点。
- 公共牌和本人手牌达到新尺寸下限。
- 九座位不与顶部栏、聊天区、下注区和屏幕安全区重叠。
- 所有座位始终存在圆形头像底板。
- 翻牌前起手牌描述正确。
- 翻牌、转牌和河牌的最佳五张牌型中文名称正确。
- 牌型计算只使用本人手牌与公共牌。

### 10.2 机器人

- 普通深筹码、弱牌和中等牌力场景不会把全下加入常规候选。
- 短筹码不足跟注时可以合法全下。
- 强牌、低有效筹码和满足阈值的激进模型可以合理全下。
- 下注尺度不会默认插值到接近全部筹码。
- 不同模型与高级参数仍产生方向正确且始终合法的动作。
- 多种固定场景和大量确定性种子下，不出现九名机器人连续无条件全下。
- 机器人公共 API 边界继续无法导入其他玩家底牌或存档最终牌面。

### 10.3 动画与结算

- 单赢家奖励动画目标正确。
- 多人平分分别移动正确金额。
- 主池和边池按事件顺序移动。
- 减少动态效果时金额和赢家完全一致。
- 动画取消不影响最终筹码、存档和资金守恒。

### 10.4 离桌

- 待开局、玩家行动中、机器人行动中、结算中和保存失败状态均有明确行为。
- 进行中确认离桌会合法弃牌并后台完成结算。
- 取消确认不改变任何状态。
- 重复确认、重试和应用恢复不会重复退款。
- 离桌后返回正确来源页面并清理牌桌状态。
- 余额、牌桌筹码、底池奖励和账本总额守恒。

### 10.5 最终验收命令

- 完整 `PokerCore` Swift Package 测试。
- RiverClub 单元测试。
- 牌桌核心 UI 流程测试。
- 离桌 UI 流程测试。
- 通用 iOS `build-for-testing`。
- `git diff --check`。
- `git status --short` 确认无意外修改。

自动化使用本机可用的最新 Pro Max 模拟器，同时保留 iPhone 16 Pro Max 真机视觉验收。

## 11. 完成标准

- 用户截图指出的七类问题均有对应修复和测试。

- 机器人不再集体无条件全下，但合理短筹码和强牌全下仍然存在。
- 单赢家、多人平分和边池都能明确显示赢家与筹码移动。
