# River Club 买入、锦标赛按钮与存档布局修复设计

日期：2026-07-17

## 1. 背景

本次修复针对 iPhone 16 Pro Max 全程横屏界面的三个已复现问题：

1. 玩家账户余额足够，但重新买入普通桌时持续提示“买入失败，请重试”。
2. 锦标赛卡片底部报名按钮的文字与背景对比度过低，看起来像不可用状态。
3. 牌局存档列表右侧的输赢金额和底池信息超出可视区域，被屏幕边缘裁切。

## 2. 根因与设计结论

### 2.1 买入失败

账户余额从 128,500 变为 108,500，说明上一笔 20,000 买入已经成功进入仍然活动的普通桌会话。应用重新启动后只显示大厅，没有把遗留会话恢复为可见牌桌，也没有先执行离桌结算。玩家再次买入时，底层会话存储拒绝同时创建第二个活动普通桌会话，因此出现失败。

已确认采用以下规则：

- 不恢复并继续遗留牌桌。
- 应用启动进入大厅时，自动安全结算遗留牌桌。
- 结算完成后把玩家剩余桌上筹码返还账户，再允许新的买入。
- 结算必须复用现有安全离桌、存档和幂等业务编号机制，不能直接删除活动会话或凭空修改余额。
- 自动结算失败时保留活动会话，显示明确的中文错误和重试入口；不得生成重复存档、重复返还筹码或允许新买入覆盖旧会话。

### 2.2 锦标赛按钮对比度

锦标赛报名按钮使用金色强调背景，但没有明确指定前景色，系统环境可能为按钮文字选择低对比度的次要颜色。

设计规则：

- 可报名状态使用金色背景和深色文字。
- 已报名状态使用深色禁用背景和清晰的浅色文字。
- 不仅依赖颜色表达状态，按钮文案继续使用“免费报名”“报名 · 数量 筹码”或“已报名”。
- 所有状态保持至少 44 点触控高度，并保留现有辅助功能标识。

### 2.3 牌局存档右侧裁切

存档页由固定侧边栏、固定 220 点筛选栏、固定标题宽度、五张公共牌和固定金额区共同组成。iPhone 16 Pro Max 横屏安全区域扣除侧边栏和内边距后，部分内容的最小总宽度大于剩余空间，导致列表行从左侧溢出并裁切右侧结果。

设计规则：

- 所有布局计算以 SwiftUI 提供的实际安全区域内容宽度为准，不使用设备截图像素或硬编码整屏宽度。
- 筛选栏宽度根据可用空间在紧凑值和标准值之间调整。
- 牌局行根据可用宽度选择紧凑或标准指标，依次收紧标题区、牌面尺寸、间距和结果区。
- 输赢金额属于核心信息，必须完整显示且不得被截断。
- 底池说明可以在空间紧张时缩小字号或换行，但不能超出右侧安全边界。
- 不用横向滚动掩盖布局问题；牌局列表只保留纵向滚动。
- 普通字号和辅助功能大字号都不能把交互控件放到安全区域之外；必要时筛选面板保持已有纵向滚动。

## 3. 组件与数据流

### 3.1 启动自动结算

`AppSession` 新增可观察的遗留牌桌结算状态，并提供启动恢复入口：

1. `AppSession` 初始化后读取 `LocalPokerStore.cashSession`。
2. 若没有活动会话，直接进入现有流程。
3. 若有活动会话，构建与该会话对应的协调器和座位展示资料。
4. 调用现有安全离桌流程：当前手牌自动弃牌、其余合法动作完成、牌局记录提交、桌上余额兑现。
5. 结算成功后清理协调器和会话内存状态，刷新账户余额并允许买入。
6. 结算失败时保持底层活动会话，展示阻塞式恢复提示；重试复用同一组结算与兑现业务编号。

新的买入入口增加前置保护：只要仍有未结算活动会话，就不创建新的买入尝试，而是提示先完成自动结算。该保护用于防止生命周期异常再次退化成笼统的交易失败。

### 3.2 锦标赛按钮展示

为锦标赛报名按钮提取纯展示状态，统一返回标题、背景色语义、前景色语义和是否可操作。视图只消费该状态，避免启用与禁用状态分别依赖 SwiftUI 的隐式着色结果。

### 3.3 存档自适应布局

`HandHistoryLayout` 继续作为纯布局计算单元，但输入改为实际内容宽度，并输出：

- 筛选栏宽度；
- 列表可用宽度；
- 标题宽度；
- 公共牌尺寸；
- 行内间距和内边距；
- 输赢结果区宽度；
- 底池说明的行数与字号模式。

`HandHistoryView` 使用顶层 `GeometryReader` 计算一次页面指标，把结果分别传给筛选面板和内容列表。`HandHistoryRow` 只按收到的指标布局，不自行猜测屏幕尺寸。

## 4. 错误处理

- 遗留会话自动结算期间禁止重复触发结算和新的买入。
- 自动结算失败时显示“上次牌桌结算失败，请重试”，并提供重试按钮。
- 自动结算错误不能被买入弹窗转换成普通的“买入失败”。
- 文件保存或兑现失败时保留原始持久化状态，继续使用现有原子事务保证。
- 锦标赛按钮和存档布局不改变赛事模拟数据、报名业务规则、存档内容或删除行为。

## 5. 测试与验收

### 5.1 单元测试

- 存在准备中、进行中或待结算的遗留普通桌会话时，启动自动结算会安全关闭会话并返还剩余筹码。
- 自动结算重试复用业务编号，不重复保存牌局或返还筹码。
- 自动结算失败时仍保留活动会话，并阻止新的买入。
- 锦标赛可报名和已报名状态输出明确且高对比的前景、背景语义。
- 存档布局在 iPhone 16 Pro Max 横屏可用宽度下，筛选栏与列表总宽度不超过容器宽度。
- 紧凑存档行的最小宽度不超过其分配宽度，结果区完整保留。

### 5.2 界面测试

- 使用遗留活动会话启动应用，自动结算后余额恢复且可成功买入新桌。
- 锦标赛报名按钮在启用和已报名状态均可清晰阅读。
- iPhone 16 Pro Max 横屏下，存档列表每行右侧输赢金额和底池说明完全位于安全区域内。
- 页面保持纵向浏览，不出现用于补救裁切的横向滚动。

## 6. 不在本次范围

- 同时保留或恢复多个普通桌会话。
- 修改锦标赛报名扣款、开赛、排名或奖励逻辑。
- 改版存档详情页或改变存档数据格式。
- 重做整体视觉主题、侧边栏导航或牌桌界面。

## 7. 完成标准

当遗留会话可在启动时安全结算、余额返还后首次买入成功，锦标赛按钮状态清晰可读，并且存档列表右侧全部核心信息在 iPhone 16 Pro Max 横屏安全区域内完整显示时，本次修复完成。
