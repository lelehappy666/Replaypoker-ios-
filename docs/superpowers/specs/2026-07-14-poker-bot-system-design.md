# River Club 公平机器人系统设计

日期：2026-07-14  
状态：设计已确认  
目标设备：iPhone 16 Pro Max  
适用范围：本地普通桌与后续单桌锦标赛

## 1. 目标

为 River Club 增加可配置、可复现且不能作弊的本地德州扑克机器人系统。机器人必须只依据自身可见信息、公开牌局状态、合法动作和允许使用的已结束历史作出合理反应，并支持三档难度、四种决策模型及全局高级参数。

本规格只覆盖机器人决策模块、全局设置模型与持久化边界。普通桌 SwiftUI 可玩闭环在本模块完成后单独设计和实施。

## 2. 范围

### 2.1 包含

- 独立纯 Swift `PokerBot` 模块。
- 三档难度：简单、标准、困难。
- 四种模型：稳健型、均衡型、激进型、自适应型。
- 规则决策与困难模式蒙特卡洛模拟的混合方案。
- 激进度、诈唬频率、跟注宽度和下注尺度参数。
- 快速、标准、自然三档思考时间。
- 已结束牌局公开历史分析开关。
- 每个机器人稳定且不超过正负 5 点的固定性格偏移。
- 全局设置的本地持久化、恢复推荐设置和版本迁移入口。
- 决策取消、超时、状态过期及保底动作。

### 2.2 不包含

- 外部大模型、在线策略服务或网络请求。
- 专业求解器、实时 GTO 求解或训练功能。
- 使用其他玩家当前底牌、剩余牌堆、洗牌种子或未来随机结果。
- 从牌局存档读取弃牌玩家最终底牌用于机器人决策或训练。
- 普通桌 SwiftUI 协调器、牌桌动画和机器人行动展示。
- 锦标赛阶段策略；本模块只预留公开上下文扩展入口。

## 3. 模块边界

新增独立 Swift Package 产品 `PokerBot`，依赖 `PokerCore`，不依赖 `PokerSession`、SwiftUI、Foundation 文件存储或系统时钟。

### 3.1 `BotObservation`

机器人唯一允许读取的牌局输入，包含：

- 当前牌局与行动版本标识。
- 机器人座位及自己的两张底牌。
- 已公开的公共牌。
- 公开座位状态、筹码、当前街投入额、弃牌与全下状态。
- 庄位、大小盲、当前街、底池和待跟注金额。
- 已公开行动历史。
- 规则引擎给出的合法动作及金额上下限。

该类型不得包含其他玩家底牌、完整牌堆、剩余牌堆、洗牌种子、检查点或未来事件。构造入口只接受 `PlayerObservation` 和公开会话元数据，不能接受完整内部牌局状态。

### 3.2 `BotSettings`

全局值对象，包含：

- `difficulty`：简单、标准、困难，默认标准。
- `model`：稳健型、均衡型、激进型、自适应型，默认均衡型。
- `aggression`：0–100，默认 50。
- `bluffFrequency`：0–100，默认 30。
- `callingWidth`：0–100，默认 50。
- `betSizing`：0–100，默认 50。
- `thinkingSpeed`：快速、标准、自然，默认标准。
- `analyzesHistory`：默认开启。
- `schemaVersion`：用于后续设置迁移。

所有解码值必须重新验证。范围外数值、未知枚举或不受支持的版本必须返回明确错误，不能静默截断。

### 3.3 `RuleBasedEvaluator`

纯函数规则评分器，负责：

- 翻牌前起手牌分组与位置权重。
- 当前成牌、听牌和牌面湿润度。
- 底池赔率、隐含风险、有效筹码和下注尺度。
- 已公开行动中的主动性与压力。
- 根据难度选择允许使用的特征集合。

### 3.4 `MonteCarloEstimator`

只在困难模式启用。它从机器人视角的未知牌集合中采样其他玩家可能底牌和未发公共牌，不能读取真实隐藏牌。

模拟次数由思考时间固定：

- 快速：800 次。
- 标准：2,000 次。
- 自然：5,000 次。

估算结果包含胜率、平局率和有效权益。相同观察、决策种子和模拟次数必须得到相同结果。

### 3.5 `BotDecisionEngine`

决策引擎组合规则评分、可选模拟结果、模型权重、全局参数和机器人固定偏移，从合法动作集合中返回一个 `BotDecision`。下注金额必须先按参数生成，再裁剪到规则引擎提供的合法范围和离散筹码单位。

### 3.6 `BotSettingsStore`

设置存储位于应用层，使用原子文件写入语义保存全局设置。读取失败不能覆盖已有文件；损坏设置返回可恢复错误，并允许用户恢复推荐设置。

当前一手开始时冻结设置快照。设置页面的修改只影响下一手，不改变正在进行的一手。

## 4. 难度行为

### 4.1 简单

- 使用当前牌力、基础位置和合法动作。
- 不计算精确底池赔率和对手范围。
- 不运行蒙特卡洛模拟。
- 在合法动作中加入受决策种子控制的少量非最优选择。

### 4.2 标准

- 使用起手牌范围、位置、底池赔率、牌面结构、有效筹码和下注尺度。
- 使用有限的公开行动特征。
- 不运行蒙特卡洛模拟。

### 4.3 困难

- 使用标准难度全部特征。
- 增加公开行动历史、对手范围估算和有效筹码压力。
- 按思考时间运行 800、2,000 或 5,000 次蒙特卡洛模拟。
- 自适应模型可读取允许的已结束公开历史摘要。

## 5. 决策模型和高级参数

### 5.1 决策模型

- 稳健型：收紧边缘入池范围，降低高风险跟注和诈唬权重。
- 均衡型：在价值下注、跟注和受控诈唬之间保持中性权重。
- 激进型：扩大开池、持续下注和加注范围，但仍受牌力与赔率约束。
- 自适应型：根据玩家已公开的已结束历史摘要调整范围估计；历史分析关闭或没有足够样本时退化为均衡型。

### 5.2 高级参数

- 激进度影响主动下注和加注倾向。
- 诈唬频率只影响具备合理弃牌权益的候选节点，不能强制无条件诈唬。
- 跟注宽度影响边缘权益与底池赔率阈值。
- 下注尺度在合法范围内调整价值下注和诈唬尺寸。

参数先改变策略权重，再由合法动作集合裁剪。任何极端设置都不能生成非法动作。

### 5.3 固定性格偏移

每个机器人根据稳定身份和设置版本生成激进度、诈唬频率、跟注宽度及下注尺度偏移，每项范围为 -5 至 +5。同一机器人在不同启动和牌局中保持相同偏移；偏移只作用于 0–100 参数并在边界内截断，不改变难度、公平接口或模拟次数。

## 6. 决策流程与并发约束

决策数据流为：

`PlayerObservation → BotObservation → 规则评分 → 困难模式模拟 → 模型与参数加权 → 合法动作裁剪 → BotDecision`

- 决策请求携带牌局标识、行动座位和状态版本。
- 计算在后台任务执行，不阻塞 SwiftUI 主线程。
- 状态版本变化、进入后台、离桌或恢复牌局时取消旧任务。
- 旧版本结果即使完成也不能提交给规则引擎。
- 计算时间与展示思考延迟分离；提前完成仍按设置选择合理延迟。
- 计算超时、取消、模拟失败或无有效候选时执行保底动作：能过牌则过牌，否则弃牌。
- 最终动作仍必须由 `PokerCore` 再次验证，机器人模块不能直接修改牌局状态。

建议展示延迟保持既有产品规格：快速 0.2–0.5 秒、标准 0.6–1.2 秒、自然 1.2–2.5 秒。延迟使用决策种子派生，保证测试可复现。

## 7. 历史分析边界

历史分析输入使用独立 `BotHistorySummary`，只包含已结束牌局的公开统计，例如入池率、主动加注率、摊牌频率、公开下注尺度和样本数量。

以下信息不得进入摘要：

- 未摊牌玩家的最终底牌，包括存档中为人工复盘保存的弃牌底牌。
- 当前进行中牌局的隐藏信息。
- 完整牌堆、随机种子和恢复检查点。

关闭历史分析时，应用层不得加载或传入任何历史摘要。

## 8. 设置界面要求

后续“设置 → 机器人设置”展示：

- 难度分段选择器。
- 决策模型分段选择器。
- 激进度、诈唬频率、跟注宽度、下注尺度滑杆及整数值。
- 思考时间选择器。
- 分析历史牌局开关。
- 恢复推荐设置按钮和二次确认。
- “设置从下一手生效”的持续说明。

设置页面只编辑草稿；用户确认保存后原子写入。保存失败时保留草稿和原设置，并显示可重试错误。

## 9. 错误处理

- 非法观察、空合法动作集合和数值溢出返回明确领域错误。
- 模拟器收到重复牌或不可能牌张数量时拒绝计算。
- 设置文件损坏时不自动覆盖，用户明确恢复推荐设置后才写入新文件。
- 机器人策略不得吞掉 `PokerCore` 的非法动作错误；协调器记录诊断并执行保底动作。
- 日志只记录决策版本、公开输入摘要、候选分数和结果，不记录未知牌样本的完整明细或任何真实隐藏状态。

## 10. 测试与验收

### 10.1 公平性

- 公开 `PokerBot` API 无法接受完整内部牌局、牌堆、检查点或其他玩家底牌。
- `BotObservation` 编码结果不包含洗牌种子、未发牌或其他玩家底牌。
- 历史摘要生成明确排除未摊牌底牌和进行中隐藏信息。

### 10.2 决策正确性

- 三档难度、四种模型和参数边界均只能返回合法动作。
- 下注和加注金额始终位于规则引擎给出的合法范围。
- 相同观察、设置、机器人身份和决策种子产生相同动作。
- 简单与标准模式不调用模拟器。
- 困难模式按快速、标准、自然分别执行 800、2,000、5,000 次模拟。
- 超时、取消、模拟失败和策略无候选均返回合法保底动作。

### 10.3 参数与持久化

- 每个高级参数覆盖 0、推荐值和 100。
- 固定性格偏移稳定且始终位于 -5 至 +5。
- 设置保存、重新打开、损坏文件、写入失败和恢复推荐设置满足原子语义。
- 当前手使用冻结设置；修改只在下一手生效。

### 10.4 属性与性能

- 大量随机合法观察下不崩溃、不越界下注、不返回非法动作。
- 未知牌采样不重复机器人底牌或公共牌。
- iPhone 16 Pro Max 目标配置下，困难模式三档模拟量均可取消且不阻塞主线程。
- 性能测试记录 800、2,000 和 5,000 次模拟的耗时分布，为后续协调器设置超时预算提供依据；本规格不通过降低既定模拟次数来掩盖性能问题。

## 11. 交付顺序

1. 公平观察类型、设置值对象和固定性格偏移。
2. 规则评分器和合法动作候选生成。
3. 可复现蒙特卡洛估算器。
4. 难度、模型与高级参数决策组合。
5. 取消、超时、过期结果和保底动作。
6. 设置持久化与后续 SwiftUI 设置界面所需应用接口。
7. 完整公平性、属性、性能与公开 API 边界测试。

本模块通过验收后，再进入“普通桌 SwiftUI 可玩闭环”规格和实施。
