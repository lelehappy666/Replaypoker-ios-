# River Club 本地功能补齐与赌场筹码动画 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 补齐普通桌本地互动、可完整游玩的单桌锦标赛、个人资料、成就和本地安全，并将筹码动画升级为流畅的赌场风格。

**Architecture:** 保留 `PokerCore` 作为唯一牌局规则真值；牌桌动画仅是由事件驱动的显示投影。锦标赛在 `PokerSession` 增加版本化本地状态和幂等账本命令，应用层使用独立协调器。资料与成就使用原子文件存储，密码使用 Keychain，Face ID 只通过 `LocalAuthentication` 验证。

**Tech Stack:** Swift 6、SwiftUI、Observation、PokerCore/PokerSession/PokerBot/PokerCoordinator、PhotosUI、LocalAuthentication、Security、XCTest/XCUITest、iOS 18+。

## Global Constraints

- 全程支持 iPhone 16 Pro Max 横屏，左右方向均不得被刘海和安全区遮挡。
- 所有文档、说明、Git 和 GitHub 提交使用中文。
- 机器人不得读取其他玩家底牌、剩余牌堆、随机种子或未来事件。
- 所有筹码仅为无现金价值的本地娱乐筹码。
- 新增存档必须支持版本迁移和原子写入；扣款、退款和派奖必须幂等。
- 动画不得修改牌局规则状态，最终金额必须以规则与账本为准。
