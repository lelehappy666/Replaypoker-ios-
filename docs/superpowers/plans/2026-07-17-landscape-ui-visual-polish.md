# River Club 横屏界面视觉统一实施计划

> **执行要求：** 直接在当前 `main` 分支实现；所有说明与提交信息使用中文；按用户要求不运行自动化测试，由用户在 Xcode 中自行验收。

**目标：** 以用户提供的图 8、图 9 为最终视觉标准，统一 iPhone 16 Pro Max 横屏下的登录、大厅、存档、个人中心和牌桌界面，并消除空白、截断、错位和遮挡。

**实现方式：** 延续现有 SwiftUI 与 `RCTheme`，用共享侧栏宽度和自适应布局代替页面内固定窄宽度。大厅改为单列推荐区加双列热门区；牌桌保留现有游戏状态与交互，仅重做展示层和操作区。背景使用与深绿、暗金界面统一的赌场场景图，并叠加可读性渐变。

**技术栈：** SwiftUI、Asset Catalog、现有 PokerCore/PokerCoordinator 展示状态。

## 全局约束

- 目标设备为 iPhone 16 Pro Max 横屏。
- 不修改德州扑克规则、机器人决策或存档业务逻辑。
- 所有金额、盲注和按钮标题必须完整显示，不使用省略号。
- 游戏大厅、锦标赛、我的牌局、个人中心统一使用相同侧栏宽度。
- 正常运行路径不展示调试组件。

## 任务

### 任务 1：统一背景与侧栏

- 登录页增加暗色河畔赌场背景，保留左侧文案和右侧品牌卡。
- 大厅使用俯视绿色牌桌背景；存档使用牌局记录氛围背景；个人中心使用会员休息区背景。
- 所有带侧栏页面统一使用 `AppSidebar.landscapePhoneWidth`，内容区域自行适配剩余宽度。

### 任务 2：重构大厅

- 顶部标题、通知和美元余额固定在同一水平区。
- 分类选择器占满内容宽度。
- 快速加入完整显示 `100 / 200`、`200 / 400`、`500 / 1,000`。
- 推荐牌桌横跨整行，人物头像和名字水平排列。
- 热门牌桌改为底部双列，顶部与推荐卡左右边缘对齐。
- 常规牌桌、已收藏、新手专区直接显示对应牌桌卡片，不再保留推荐区空白。

### 任务 3：重构牌桌视觉

- 对照图 8 使用圆形真人头像、头像绑定名字、独立两张牌背和筹码余额。
- 中心保留五张公共牌槽、底池筹码和当前手牌说明。
- 玩家下注筹码显示在各自前方；操作区保持右下角独立卡片。
- 操作按钮完整显示跟注与加注金额，允许金额文本缩放但不省略。

### 任务 4：错误与结算状态

- 操作失败提示移到操作区上方的独立位置，不覆盖“本手牌已结束”和下一手按钮。
- 错误提示按钮保持完整触控尺寸并可关闭、重试。

### 任务 5：交付

- 检查代码差异、资源引用和中文提交信息。
- 不运行测试或模拟器，由用户自行测试视觉和交互。
