# 牌桌动画与恢复复审修复实施计划

> **供代理执行：** 本计划在当前会话内按测试驱动方式逐项执行；项目明确禁止本轮派发子代理。

**目标：** 让每个牌桌动画事件拥有独立、单调递增的发布序号，并让 suspended 状态下的本地操作重试恢复同一手牌而非重复发送原意图。

**架构：** `stateVersion` 继续只标识牌局业务状态，新增 `animationSequence` 只在成功发布动画状态时受检递增，并由所有投影和状态复制路径显式携带。交互层保留失败意图，通过统一重试方法按当前阶段选择 `resume()` 或重发原意图，成功后清错，恢复失败时保留意图并更新中文错误。

**技术栈：** Swift 6、SwiftUI、Observation、Swift Testing、XCTest、XcodeGen。

## 全局约束

- 所有文档、计划、报告和交付说明使用中文。
- 所有 Git 提交及 GitHub 提交信息使用中文。
