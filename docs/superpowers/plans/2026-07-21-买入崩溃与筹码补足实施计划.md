# 买入崩溃与筹码补足实施计划

> **面向代理式开发者：** 必须使用 `superpowers:executing-plans` 按任务执行，每个步骤使用复选框跟踪。

**目标：** 修复买入面板零长度滑杆崩溃，并将新旧账户的娱乐筹码安全、幂等地补足到 1,000,000。

**架构：** 买入界面通过纯状态属性决定是否创建 SwiftUI `Slider`，避免把非法范围交给控件。筹码补足通过现有可审计账本新增幂等收据，并在生产存档打开后执行固定版本的一次性迁移。

**技术栈：** Swift 6、SwiftUI、XCTest、Swift Testing、PokerSession 本地 JSON 持久化。

## 全局约束

- 所有说明文档和 Git 提交使用中文。
- 不删除牌局存档，不改动未结算桌上筹码。
- 新账号初始筹码和现有低余额账号的补足目标均为 1,000,000。
- 同一补足版本只能产生一条账本收据。
