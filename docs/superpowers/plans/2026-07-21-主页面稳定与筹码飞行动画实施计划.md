# 主页面稳定与筹码飞行动画实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 消除四个主页面切换时的整体位移，并为下注、归池、退回和派彩增加分层筹码飞行动画，同时使右上余额组件与确认效果图一致且不遮挡手牌。

**Architecture:** 主页面外壳统一计算固定导航栏框和右侧内容框，业务页面只能在内容框内布局。牌桌动画模型把单一进度转换成四个错峰筹码簇的进度，牌桌视图根据事件方向绘制临时飞行层；右上余额使用独立的立体筹码组件。

**Tech Stack:** Swift 6、SwiftUI、Observation、XCTest、Xcode iOS 模拟器。

## Global Constraints

- 所有说明文档和 Git 提交信息使用中文。
- 目标设备为 iPhone 16 Pro Max 横屏，并兼容现有横屏参考画布。
- 系统开启“减少动态效果”时仍保留筹码分批飞动，仅缩短时间和减小弧度。
- 不修改牌局规则、机器人身份、金额计算和结算事件顺序。
- 不增加大规模 UI 自动化测试，仅增加少量纯逻辑测试并执行编译检查。

### Task 1: 固定四个主页面外壳

**Files:**

- Test: `RiverClubTests/AppSessionTests.swift`

**Interfaces:**

- Produces: `SidebarPageShellLayout.frames(in:) -> (sidebar: CGRect, content: CGRect)`，供公共外壳稳定定位。

- [ ] **Step 1: 编写失败的固定框测试**

在 `AppSessionTests.swift` 增加测试，使用同一 `CGSize(width: 932, height: 430)` 连续计算四次，断言导航栏框和内容框完全相同，并断言内容框宽度等于画布扣除左右边距、导航栏和间距后的值。

- [ ] **Step 2: 运行测试确认失败**

Run:

```bash
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer xcodebuild -project RiverClub.xcodeproj -scheme RiverClub -destination 'platform=iOS Simulator,name=iPhone 17 Pro Max' -only-testing:RiverClubTests/AppSessionTests/testSidebarShellUsesStableExplicitFrames test
```

Expected: FAIL，原因是 `SidebarPageShellLayout` 尚不存在。

- [ ] **Step 3: 实现固定画布计算**

在 `AppRootView.swift` 增加 `SidebarPageShellLayout`，按以下公式返回框：

```swift
let horizontalInset = AppSidebar.minimumSafeInset
let verticalInset = AppSidebar.shellVerticalPadding
let sidebar = CGRect(
    x: horizontalInset,
    y: verticalInset,
    width: AppSidebar.landscapePhoneWidth,
    height: max(canvas.height - verticalInset * 2, 0)
)
let contentX = sidebar.maxX + AppSidebar.contentGap
let content = CGRect(
    x: contentX,
    y: verticalInset,
    width: max(canvas.width - horizontalInset - contentX, 0),
    height: sidebar.height
)
```

`SidebarPageShell` 改用 `ZStack` 和 `.position` 分别放置导航栏与右侧内容，不再让 `HStack` 根据业务内容理想尺寸决定整体宽度。背景层保持满画布，外壳继续禁用隐式切换动画。

运行 Step 2 命令，Expected: PASS。

### Task 2: 增加四组错峰筹码飞行动画

**Files:**
- Modify: `RiverClub/Features/Table/TableInteractionState.swift`

**Interfaces:**
- Consumes: `TableAnimationEvent` 的 `moveCommitmentToPot`、`returnUncalledBet` 和 `awardPot`。
- Produces: `TableAnimationPresentation.chipFlightProgress(at:reduceMotion:)`、`PokerTableLayout.chipFlightPosition(from:to:progress:arcOffset:)`。

- [ ] **Step 1: 编写失败的错峰进度与弧线路径测试**

增加两项纯逻辑测试：

```swift
XCTAssertEqual(presentation.chipFlightProgress(at: 0, reduceMotion: false), 1)
XCTAssertGreaterThan(
    presentation.chipFlightProgress(at: 0, reduceMotion: false),
    presentation.chipFlightProgress(at: 3, reduceMotion: false)
)
```

以及路径端点测试：进度 `0` 等于起点，进度 `1` 等于终点，中间点带有非零弧线偏移。

- [ ] **Step 2: 运行目标测试确认失败**

Run:

```bash
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer xcodebuild -project RiverClub.xcodeproj -scheme RiverClub -destination 'platform=iOS Simulator,name=iPhone 17 Pro Max' -only-testing:RiverClubTests/PokerTableInteractionTests -only-testing:RiverClubTests/PokerTableLayoutTests test
```

Expected: FAIL，原因是错峰进度与弧线路径接口尚不存在。

- [ ] **Step 3: 实现动画展示逻辑**

`TableAnimationPresentation` 保留事件总进度，并为四个筹码簇使用 `[0, 0.08, 0.16, 0.24]` 的归一化延迟。普通模式使用完整延迟，减少动态效果模式将延迟缩短一半，但仍返回不同进度。

`PokerTableLayout.chipFlightPosition` 使用二次贝塞尔插值：起点和终点保持精确，中间控制点沿路径法线偏移 `arcOffset`。

- [ ] **Step 4: 绘制三种方向的临时筹码层**

在 `PokerTableView` 中用一个 `chipFlightLayer` 处理：

- `moveCommitmentToPot`：下注框到中央底池。
- `returnUncalledBet`：下注框到玩家座位。
- `awardPot`：中央底池到赢家座位。

每次绘制四个小型筹码簇，分别使用 `[-8, -3, 4, 9]` 的弧线偏移、轻微不同缩放和旋转。金额文字只显示一次，不随四组重复。金额为零或缺少位置时返回空视图，不阻塞牌局。

- [ ] **Step 5: 调整动画持续时间**

普通模式中移动筹码使用约 `0.62` 秒，派彩使用约 `0.78` 秒；减少动态效果模式分别使用约 `0.36` 秒和 `0.46` 秒。重置任务在动画末尾后增加约 `0.08` 秒，确保最后一组筹码完整到达。

运行 Step 2 命令，Expected: PASS。
